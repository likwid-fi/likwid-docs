---
title: "特殊情况下的流动性耗尽处理"
description: "当池子中的流动性耗尽，特别是某一侧代币被完全抽干时，系统会引入替代机制以尽量维持交换能力。"
---

当池子中的流动性耗尽，尤其是某一侧资产在 swap 池中已经无法直接提供时，系统会引入一套机制，以便在极端情况下仍然尽量维持交换能力。

**在流动性耗尽时进行交换**

* **流动性枯竭**：当 **swap 池** 中某个资产的可用流动性被完全消耗时，系统允许继续通过 **lA tokens** 来完成交换。这些代币是从借贷池中派生出来的生息型资产表示。
* **lA Tokens**：**lA token** 是 **Token A** 的一种代币化形式，表示 Token A 在借贷池中的生息版本。它们作为借贷机制的一部分被铸造出来，并且在 swap 池中 Token A 流动性耗尽时，可以临时代替 Token A 使用。
  * **lA token** 可以像 Token A 一样参与交换，从而避免用户在流动性不足时完全无法交易。
  * lA token 仍然会继续从借贷池中累积利息，并可在池子流动性恢复后赎回。

**重新转换回 Token A**

* **赎回 lA Tokens**：当 Token A 在 swap 池中的流动性恢复后，持有 lA token 的用户可以按照当前 Token A 与 lA 之间的兑换率随时赎回为 Token A。
* **动态调整**：Token A 与 lA 的兑换率会动态变化，以确保用户按当时正确的价值赎回，其中也包含了借贷池累计产生的利息。

**需要注意的点**

* **用户灵活性**：使用 lA token 作为替代形式，可以让用户在流动性紧缺时仍然完成交换，改善使用体验并减少交易失败。
* **系统效率**：这种机制让系统在高需求阶段依然能够继续运行，同时为池子恢复流动性争取时间。
