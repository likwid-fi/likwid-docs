### **1. 问题定义：流动性撤出风险**

突发且大规模的流动性撤出会对去中心化协议造成系统性冲击：

* **级联清算**：流动性骤减会造成价格错位，并触发原本健康度边缘的仓位进入抵押不足状态
* **套利脆弱性**：恶意参与者可以利用短时间的流动性缺口，在几乎无风险的情况下抽取利润

### **2. 机制设计：分层解锁框架**

一种动态、多周期的解锁系统，会通过数学约束来限制流动性退出速度。

**2.1 核心锁定参数**

* **锁定周期（m）**：固定数量的连续时间周期，例如 `m = 8` 个按小时计算的时间段
* **基础解锁额度**：每个周期的标准释放量为 `Q = T / m`，其中 `T` 为总锁定价值
* **触发要求**：用户需要在每个周期中主动发起提款请求

**2.2 自适应提款规则**

第 `i + 1` 个周期可以释放多少，取决于第 `i` 个周期的提款行为。

| **第 i 个周期的提款比例（R\_i）** | **第 i+1 个周期的可分配额度（A\_i+1）** |
| --------------------------------- | ---------------------------------------- |
| `R_i < 20%`（退出压力低）         | `A_i+1 = Q`（基础额度）                  |
| `R_i ≥ 20%`（退出压力高）         | `A_i+1 = (Q - W_i) + Q`（滚动结转）      |

其中：

* `W_i` = 第 `i` 个周期实际提款量
* `R_i = W_i / A_i`（提款量占可用额度的比例）

### **5. 结论**

这一分层解锁机制通过数学方式约束流动性撤出速度，同时仍然保留正常用户的提款能力。系统会根据实时提款压力和市场状态动态调整解锁额度，从而在不牺牲用户自主权的前提下显著提升协议稳定性。它的参数化设计（例如周期数 `m` 与触发阈值）也让协议可以根据自身风险偏好进行校准。
